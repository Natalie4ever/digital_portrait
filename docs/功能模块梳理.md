# 员工数字画像系统 — 功能模块梳理

本文档梳理系统现有功能模块及其作用，便于按模块逐一细化和修改。

---

## 一、项目概览

| 项目 | 说明 |
|------|------|
| 系统名称 | 员工数字画像系统 |

## 三、功能模块一览（按模块）

以下每个模块均可单独进行细化和修改。

---

### 模块 1：认证 (Auth)

**作用**：登录、JWT 鉴权、当前用户信息、修改密码；前端维护登录态与自动登出。

| 层级 | 路径 | 说明 |
|------|------|------|
| 后端路由 | `backend/app/routers/auth.py` | 前缀 `/api/auth`：登录、当前用户、修改密码 |
| 后端鉴权 | `backend/app/auth.py` | JWT 签发/校验、PBKDF2 密码哈希、`get_current_user`、`get_current_admin`、`can_access_user`（组长看本组预留） |
| 前端页面 | `frontend/src/pages/Login.jsx` | EHR 号 + 密码登录，成功后写 token 并跳转 |
| 前端页面 | `frontend/src/pages/ChangePassword.jsx` | 个人修改密码 |
| 前端状态 | `frontend/src/contexts/AuthContext.jsx` | `AuthProvider`：user、loading、refreshUser、logout；`useAuth()`；依赖 `api.get('/auth/me')` |
| 前端 API | `frontend/src/api.js` | `login`、`getCurrentUser`、`changePassword`；request 带 Bearer token，401 清 token 并跳转登录 |
| 布局行为 | `frontend/src/components/Layout.jsx` | 顶栏、菜单、用户下拉（修改密码/退出）、**1 小时无操作自动登出** |

**可细化方向**：登录失败次数限制、验证码、记住我、多端登出、token 刷新、组长权限具体实现等。

---

### 模块 2：个人档案 (Profile)

**作用**：员工全维度档案的查看与编辑，包含基础信息及多条子表（政治、学历、家庭、简历、奖惩、资格、成果、语言、通讯、技能标签等）。

| 层级 | 路径 | 说明 |
|------|------|------|
| 后端路由 | `backend/app/routers/profile.py` | 前缀 `/api/profile`：档案完整 CRUD；支持 `/me` 与 `/by-ehr/{ehr_no}`（组长看组员） |
| 后端模型 | `backend/app/models.py` | `Profile` 及子表：`PoliticalInfo`、`EducationInfo`、`FamilyInfo`、`ResumeInfo`、`RewardInfo`、`QualificationInfo`、`AchievementInfo`、`LanguageInfo`、`ContactInfo`、`ProfileSkillTag` |
| 后端 Schema | `backend/app/schemas.py` | 档案及各子表的请求/响应 Pydantic 模型 |
| 前端页面 | `frontend/src/pages/Profile.jsx` | 档案查看/编辑（基础信息与各子表，子表支持多条、按时间排序） |
| 前端 API | `frontend/src/api.js` | `getProfileMe`、子表 CRUD、contact、skill-tags 等 |

**可细化方向**：字段校验规则、子表排序/分页、历史版本、导出 PDF/Excel、组长只读查看逻辑、档案审批流程等。

---

### 模块 3：用户管理 (User Management)

**作用**：仅管理员可用；用户增删改查、分页筛选、重置密码、启用/禁用（软删除）、Excel 批量导入。

| 层级 | 路径 | 说明 |
|------|------|------|
| 后端路由 | `backend/app/routers/users.py` | 前缀 `/api/admin/users`：用户列表(分页/筛选)、增删改、重置密码、批量 Excel 导入 |
| 后端模型 | `backend/app/models.py` | `User`（含角色、组别、启用状态等） |
| 后端 Schema | `backend/app/schemas.py` | 用户 CRUD、批量导入等请求/响应模型 |
| 前端页面 | `frontend/src/pages/AdminUsers.jsx` | 用户列表、增删改、重置密码、Excel 导入 |
| 前端 API | `frontend/src/api.js` | 管理员用户相关接口（含 `batchImportUsers`） |

**可细化方向**：角色/组别字段含义与约束、导入模板与错误提示、批量操作审计、禁用用户的登录拦截等。

---

### 模块 4：操作日志 (Operation Logs)

**作用**：记录关键操作（如登录、改密等）；仅管理员可查看，支持分页与按用户、动作筛选。

| 层级 | 路径 | 说明 |
|------|------|------|
| 后端路由 | `backend/app/routers/operation_logs.py` | 前缀 `/api/admin/operation-logs`：日志列表(分页/按用户、动作筛选) |
| 后端写入 | `backend/app/operation_log.py` | `log_operation(db, user_id, action, resource, detail, ip)`，被 auth、users 等调用 |
| 后端模型 | `backend/app/models.py` | `OperationLog` |
| 后端 Schema | `backend/app/schemas.py` | 操作日志列表、筛选参数等 |
| 前端页面 | `frontend/src/pages/AdminLogs.jsx` | 日志列表与筛选 |
| 前端 API | `frontend/src/api.js` | 操作日志列表接口 |

**可细化方向**：记录更多操作类型、保留时长与归档、导出、敏感信息脱敏等。

---

### 模块 5：技能标签模板 (Skill Tags)

**作用**：预定义技能标签由管理员维护；用户在个人档案中选择或补充自定义标签。

| 层级 | 路径 | 说明 |
|------|------|------|
| 后端路由 | `backend/app/routers/skill_tags.py` | 前缀 `/api/skill-tags`：模板列表；管理员可增删模板 |
| 后端模型 | `backend/app/models.py` | `SkillTagTemplate`、`ProfileSkillTag`（档案与标签关联） |
| 后端 Schema | `backend/app/schemas.py` | 技能标签模板、档案技能标签等 |
| 前端页面 | `frontend/src/pages/AdminSkillTags.jsx` | 管理员维护预定义标签 |
| 前端 | `frontend/src/pages/Profile.jsx` + `api.js` | 档案中技能标签块与接口 |

**可细化方向**：标签分类、排序、使用统计、是否允许用户自定义及审核等。

---

### 模块 6：数据校验 (Validators)

**作用**：统一校验规则（身份证、手机号、电话、日期等），供档案等接口使用。

| 层级 | 路径 | 说明 |
|------|------|------|
| 后端 | `backend/app/validators.py` | 身份证、手机号、电话、日期格式等校验函数 |

**可细化方向**：更多证件类型、邮箱、地址规范、与前端校验规则同步等。

---

### 模块 7：数据库与配置 (Database & Config)

**作用**：数据库连接、会话管理、应用配置、请求/响应模型集中定义。

| 层级 | 路径 | 说明 |
|------|------|------|
| 数据库 | `backend/app/database.py` | Async SQLAlchemy：engine、AsyncSessionLocal、get_db、Base |
| 配置 | `backend/app/config.py` | Pydantic Settings：DATABASE_URL、SECRET_KEY、ALGORITHM、ACCESS_TOKEN_EXPIRE_MINUTES、DEFAULT_PASSWORD；支持 `.env` |
| Schema | `backend/app/schemas.py` | 登录/改密、用户、档案各子表、技能标签、操作日志等 Pydantic 模型 |
| 初始化 | `backend/init_db.py` | 建表 + 可选创建默认管理员（ehr=admin，密码=1234567） |

**可细化方向**：多环境配置、迁移脚本、连接池与超时、敏感配置脱敏等。

---

### 模块 8：健康检查与入口 (Health & Entry)

**作用**：应用入口、路由挂载、CORS、启动时建表、健康检查接口。

| 层级 | 路径 | 说明 |
|------|------|------|
| 后端入口 | `backend/app/main.py` | FastAPI 应用、CORS、挂载各路由、启动建表、`GET /api/health` 返回 `{"status":"ok"}` |
| 前端入口 | `frontend/src/main.jsx` | 挂载 App 到 `#root` |
| 前端路由 | `frontend/src/App.jsx` | BrowserRouter + AuthProvider，所有 Route 与 PrivateRoute |

**可细化方向**：就绪/存活分离、依赖检查（DB 等）、前端路由懒加载与权限矩阵等。

---

### 模块 9：前端布局与导航 (Layout & Navigation)

**作用**：顶栏、侧栏/菜单、用户下拉、子路由出口、无操作自动登出。

| 层级 | 路径 | 说明 |
|------|------|------|
| 布局 | `frontend/src/components/Layout.jsx` | 顶栏、菜单（首页/我的档案/管理员菜单）、用户下拉（修改密码/退出）、1 小时无操作自动登出、`<Outlet />` |

**可细化方向**：菜单权限与角色、面包屑、主题/布局配置、移动端适配等。

## 六、建议的细化与修改顺序

1. **认证** — 登录与权限是基础，建议先稳定（含组长权限是否在本期实现）。
2. **数据库与配置** — 建表、迁移、环境分离，便于后续模块安全改动。
3. **数据校验** — 与档案、用户等强相关，提前统一可减少返工。
4. **个人档案** — 核心业务，字段与子表多，适合分批次细化。
5. **用户管理** — 依赖角色与权限，在认证与档案之后调整更清晰。
6. **操作日志** — 在关键操作稳定后再扩展记录范围与策略。
7. **技能标签** — 相对独立，可按产品需求单独迭代。
8. **前端布局与导航** — 与权限、菜单配置一起做，避免重复改路由。
