## 二、核心测试项

### 🔴 测试项 T2.2（核心）：调组后原组长看不到 A

**目的：** 验证组长权限基于调换历史 `leave_date IS NULL`，调离后无权限

**前置条件：**
- A 原属于「测试一组」（由「测试一组」组长管辖）
- 管理员已登录

**操作步骤：**

1. 用管理员登录 → 「用户管理」→ 找到 A → 点击「调组」按钮
2. 弹窗显示「当前组别：测试一组」
3. 调入组别选择「测试二组」（新组）
4. 调组时间默认当前 → 原因：测试
5. 点击「确认调组」→ 提示「已成功将 A 调至 测试二组」
6. 列表中 A 的组别变为「测试二组」

7. **退出登录，用「测试一组」组长登录**

8. 进入「档案管理」→ 应该**看不到 A**（只能看到「测试一组」成员）

9. 尝试通过 URL 直接访问 A 的档案：
   - 输入 `http://localhost:5173/admin/profile-view/A 的 EHR 号`
   - 应显示**「无权限」**或**空内容**

10. **退出，用「测试二组」组长登录**

11. 进入「档案管理」→ **应能看到 A**

12. 查看 A 的档案 → 正常显示

**✅ 通过标准：** 调组后原组长确实看不到 A，新组长能看到

7. 历史一条：调出组别=原组别，状态=「已离开」

**✅ 通过标准：** 历史记录正确写入，状态 Tag 正确

## 五、常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 调组后原组长仍能看到 | `_can_access_with_db` 缓存或迁移未跑 | 确认迁移脚本已跑 + 后端重启 |
| 「调入组别」不允许输入新值 | 浏览器缓存 | 硬刷新（Ctrl+Shift+R） |
| 历史列表为空 | 迁移脚本未回填 | 跑 `python backend/migrate_step2.py` |
| 操作日志无 transfer_user | 写日志失败 | 查看后端日志 |
| 调组后用户列表不刷新 | 前端没 reload | 手动刷新页面 |
