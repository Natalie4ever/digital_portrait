## 二、测试项 1：应急先锋队（员工可编辑）

**目标：** 验证员工可自行开启/关闭应急先锋队

**操作步骤：**

1. 用普通员工（0000002）登录
2. 进入「我的档案」→ 找到「通讯信息」卡片
3. 找到「应急先锋队」开关
4. 点击开启 → 提示「已加入应急先锋队」
5. 刷新页面 → 开关保持开启
6. 切换到 `我的档案` → 顶部状态：开关为「已加入」

**✅ 通过标准：** 开关正常切换，保存后刷新数据不丢失

---

## 三、测试项 2：应急先锋队（用户管理列表 - 管理员）

**目标：** 验证管理员可在用户管理列表标记任意人

**操作步骤：**

1. 用管理员（0000001）登录
2. 进入「用户管理」
3. 找到任一员工 → 点击「应急先锋队」Tag
4. 弹出确认框「确定将 XX 标记为应急先锋队成员？」
5. 点击「确定」→ 提示「已标记为应急先锋队」
6. 再次点击 → 取消
7. 在筛选区选择「应急先锋队 = 是」→ 查询 → 只显示标记过的员工
8. 筛选「应急先锋队 = 否」→ 只显示未标记的

**✅ 通过标准：** 列表可点击切换，筛选可正常工作

---

## 四、测试项 3：应急先锋队（档案管理列表 - 组长/管理员）

**目标：** 验证组长只能标记本组，管理员可标记任意

**操作步骤：**

1. 管理员进入「档案管理」
2. 列表中找到「应急先锋队」列（红色/灰色 Tag）
3. 点击 Tag 切换 → 列表实时刷新
4. 筛选「应急先锋队 = 是」→ 只显示标记过的
5. 退出，用组长（0000003）登录（如果该 EHR 对应某组组长）
6. 进入「档案管理」→ 只能看到本组成员
7. 点击本组成员 Tag → 可切换
8. 尝试访问其他组员 → 越权提示

**✅ 通过标准：** 组长只能标记本组，越权操作被后端拦截

---

## 五、测试项 4：发展意向 1:1（4 个部分折叠面板）

**目标：** 验证发展意向整合为 1:1 单表，4 个部分折叠面板

**操作步骤：**

1. 用普通员工（0000002）登录
2. 进入「我的档案」→ 滚动到**最下方**
3. 找到「📋 发展意向」卡片
4. 点击「填写」按钮
5. 4 个折叠面板依次展开/填写：
   - **第一部分（职业发展方向）**：
     - 发展序列偏好：选择「管理发展路径」
     - 短期目标：填写「1年内完成 PMP 认证」
     - 中长期目标：填写「3年内成为团队主管」
   - **第二部分（能力提升与学习需求）**：
     - 核心能力提升：选择「沟通与影响力」「流程与项目管理」
     - 学习方式：选择「外部认证课程」
     - 希望学习课程：填写「PMP、CFA」
   - **第三部分（实践机会意向）**：
     - 是否愿意轮岗：选择「是」
     - 感兴趣部门：填写「总行风控部」
     - 项目参与意向：选择「数字化转型」
   - **第四部分（其他补充）**：
     - 其他补充：填写「希望参与数字化转型项目」
6. 点击「保存」→ 提示「发展意向已保存」
7. 刷新页面 → 数据完整保存
8. 点击「编辑」修改部分内容 → 保存 → 验证
9. 点击「取消」→ 数据回到保存前

**✅ 通过标准：** 4 个部分可正常折叠展开、保存、编辑、取消

## 十一、常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 发展意向卡片空白 | 未填写 | 点击「填写」 |
| 折叠面板默认收起 | 设计如此 | 点击 panel 展开 |
| 应急先锋队 Tag 点击无反应 | 浏览器卡顿 | 刷新页面重试 |
| 列表筛选不生效 | EHR 号或筛选条件未填 | 先选筛选再查询 |
| 数据保存失败 | 后端 500 | 查看后端日志 |
